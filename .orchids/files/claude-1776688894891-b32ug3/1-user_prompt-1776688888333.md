**PROMPT START**

Build a full-stack, production-ready SaaS-style web application called **Meridiem Client Portal** for a company named **Meridiem Global**.

This is a **client-facing workforce management and billing platform**, not just a dashboard.

The system must include authentication, role-based access, database models, API structure, and integration-ready architecture.

The UI must be **mobile-first (iPhone optimized)** while still delivering a strong desktop experience.

---

## 🧠 CORE PURPOSE

Clients log in to:

* View their remote workforce
* Track hours and performance (via Hubstaff)
* View screenshots (via Hubstaff)
* Monitor billing and deposit status
* Pay invoices and enable AutoPay
* Request additional hires or replacements
* Submit support requests, complaints, and feedback
* Interact with an AI assistant

Admins manage all client accounts, support requests, hiring pipelines, and billing visibility.

---

## 👥 USER ROLES

### Admin

* Full system access
* View all clients
* Edit records
* Internal notes
* Audit logs
* “View as Client” mode

### Client Users (multiple per company)

Roles:

* client_owner
* client_billing
* client_viewer

Permissions:

* Billing users can pay invoices and manage payment settings
* Viewers have read-only access
